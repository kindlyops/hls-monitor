Recommended on iPhone, iPad & Mac

HLSMonitor — the native app

A native app for Apple platforms that measures HLS delivery — segment timing, playback stalls, bitrate switching, and audio loudness — and exports a PDF report. No browser CORS limits, because it plays the stream itself rather than reading another page's network traffic.

Any browser, zero install

Monitor page

For Android and any other browser: open the monitor page, paste an .m3u8 URL, and it fetches the playlist and segments itself on the standard HLS cadence, measuring every request — the same dashboard as the extension, no install required.

Works only on CORS-enabled streams. A web page may read another origin's responses only if that server sends Access-Control-Allow-Origin, and many streams don't. If the monitor reports a CORS error, that stream can't be measured from any browser page — that's a browser rule, not a bug. Use the native app above (Apple devices) or the Chrome extension on desktop for those; both watch the stream directly and need no CORS. If you control the stream's origin, adding Access-Control-Allow-Origin: * to playlists and segments fixes it (the same header hls.js-based web players require).
